# SMILES-2026 Signal Interference Cancellation Solution

## Reproducibility

My repository does not need any additional libraries other than initial repository. To prepare environment:

```bash
git clone https://github.com/On-Point-RND/SMILES-2026-Signal.git
cd SMILES-2026-Signal

python -m venv .venv
source .venv/bin/activate

python -m pip install --upgrade pip
python -m pip install numpy scipy gdown
```

To run the code:

```bash
uv run python applicant_solution.py
```

## Main idea

The goal is to obtain a clean, composite signal `rx_hat` by subtracting two identifiable noise components from the original `rx`:

1. TX-dependent nonlinear interference `F_c(TX)` - generated by the device own transmission channels.
2. `R1` is noise that cannot be explained by the transmitter.

Final solution formula:

```text
rx_hat = rx - tx_total - r1_total
```

where:

- `tx_total` — estimate of the TX-dependent nonlinear component;
- `r1_total` — estimate of the external R1 component;
- `rx_hat` — final signal after interference suppression.

The main idea is not to aggressively zero out the energy in the score-band at any cost, but to ensure that the removed component remains valid from the perspective of formal verification: it must be well-explained as the sum of a TX-driven model and a R1 model.

The final solution is implemented in `applicant_solution.py`.

Key changes compared to the baseline:

1. Normalization of TX channels by RMS power has been added.
2. The `fit_tx_prediction(...)` helper is used to estimate the TX-driven nonlinear component.
3. An estimate of the external R1 noise has been added on top of the baseline.
4. The TX-driven and R1 components are estimated iteratively rather than just once.  3 iterations yielded the best result

After subtracting the TX component, a residual remains; it is in this residual that we search for the external interference `R1`. By definition, it is spatially coherent: the same source is visible in all four RX channels, albeit with different amplitudes and phases.

That is why I use the R1 model. I find a single common temporal signal that best explains the similar portion across all RX channels.

The rank-1 component is estimated only in the score-band. The metric is calculated not across the entire spectrum, but within a specific band. Therefore, the model focuses specifically on the part of the signal that affects the score


The TX component and the external rank-1 component interfere with each other during estimation.

If we estimate the TX interference on the original `rx`, there is still external interference in the signal. It can slightly distort the TX regression. If we then estimate the rank-1 on the residual, pieces of TX interference may remain there.
Therefore, an iterative scheme is used:


1. First, find the external common interference given the current TX estimate;
2. Then, refine the TX component again given the current external interference;
3. Repeat several times.

In the final solution, `n_iter = 3` is used. This is sufficient: after three iterations, the result is already nearly stable, and further iterations degrade the result.

## Unsuccesful experiments

I also tested a more complex idea: replacing the time-domain ridge regression with a frequency-domain solver.

The reasoning was as follows:

- In the frequency domain, the problem can be solved separately for each frequency bin;
- this seems more flexible and requires less manual selection of lags.

The first version did indeed significantly reduce the energy in the score band:

```text
[iter 1/3] total band removed: 12.82 dB
[iter 2/3] total band removed: 12.86 dB
[iter 3/3] total band removed: 12.78 dB
```

But the result was invalid:

```text
INVALID: explainability 0.885 < 0.95
Metric [yours]: 0.00 dB
```

The solver removed energy, but the removed TX portion was no longer sufficiently explainable by the scorer’s official TX model.

Then I tried to make the FD approach more compatible: use the frequency-domain solver only to find the coefficients, but synthesize the signal through the same lag structure as the scorer. This came closer to a valid solution, but still failed the test:

```text
INVALID: explainability 0.939 < 0.95
Metric [yours]: 0.00 dB
```

After analysis, it became clear that the scorer trains the explainability model only on a fixed slice of the data:

```text
MODEL_SUBSET = slice(20_000, 220_000)
```

However, my FD regression was initially built on the entire signal. Therefore, the coefficients were optimized for a different task and did not match what the scorer later attempted to reconstruct.
